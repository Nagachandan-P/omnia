# Log Collector File-Level Log Matrix

| File Name | What It Does | Target Functional Groups (Cluster Nodes) | Logs Collected (Type) | Collected From (Source Location) | Why These Logs Are Useful |
|---|---|---|---|---|---|
| `omnia/log_collector/collect.yml` | Top-level play orchestration by stage/host group (`setup`, `prepare`, `k8s_master`, `k8s_worker`, `slurm_ctl`, `slurm_node`, `login_node`, `login_compiler_node`, `bundle`). | All: `k8s_masters`, `k8s_workers`, `slurm_controllers`, `slurm_nodes`, `login_nodes`, `login_compiler_nodes` | No direct log fetch. Triggers all collection stages. | Host groups: `k8s_masters`, `k8s_workers`, `slurm_controllers`, `slurm_nodes`, `login_nodes`, `login_compiler_nodes`. | Ensures full end-to-end collection flow and resilient execution (`ignore_unreachable: true` for node collection plays). |
| `omnia/log_collector/roles/log_collector/tasks/main.yml` | Thin router that includes stage-specific task files. | N/A | No direct log fetch. | N/A | Keeps role maintainable and stage-driven. |
| `omnia/log_collector/roles/log_collector/tasks/setup.yml` | Initializes runtime facts and log path variables for all node types. | N/A | No direct log fetch. | Loads values from role vars into runtime facts. | Standardizes shared variables for all later collection tasks. |
| `omnia/log_collector/roles/log_collector/tasks/prepare.yml` | Parses PXE CSV, builds dynamic inventory groups from `FUNCTIONAL_GROUP_NAME`, creates local workspace directories. | All: `k8s_masters` (`^service_kube_control_plane`), `k8s_workers` (`^service_kube_node`), `slurm_controllers` (`^slurm_control_node`), `slurm_nodes` (`^slurm_node`), `login_nodes` (`^login_node`), `login_compiler_nodes` (`^login_compiler_node`) | No direct log fetch. | PXE mapping file: `pxe_mapping_file_path` (default: `/opt/omnia/input/project_default/pxe_mapping_file.csv`). | Ensures nodes are targeted correctly (including login/login-compiler) and local destination paths exist. |
| `omnia/log_collector/roles/log_collector/tasks/k8s_logs.yml` | Collects Kubernetes control-plane and worker logs. | `k8s_masters` (`stage: k8s_master`), `k8s_workers` (`stage: k8s_worker`) | Container/runtime logs, pod logs, networking logs, OS/auth/system logs, package/bootstrap logs. | **K8s Master dirs:** `/var/log/containers/`, `/var/log/pods/`, `/var/log/calico/`, `/var/log/crio/`, `/var/log/chrony/`, `/var/log/private/`, `/var/log/rhsm/` ; **K8s Master files:** `/var/log/cloud-init.log`, `/var/log/cloud-init-output.log`, `/var/log/dnf.log`, `/var/log/dnf.rpm.log`, `/var/log/dnf.librepo.log`, `/var/log/hawkey.log`, `/var/log/secure`, `/var/log/spooler` ; **K8s Worker files:** `/var/log/cloud-init.log`, `/var/log/cloud-init-output.log`, `/var/log/spooler`, `/var/log/messages`, `/var/log/cron`, `/var/log/secure`, `/var/log/maillog`, `/var/log/lastlog`, `/var/log/btmp`. | Troubleshoots Kubernetes control-plane/worker issues: pod lifecycle failures, CNI/network problems, container runtime errors, node boot/provisioning issues, OS/security events. |
| `omnia/log_collector/roles/log_collector/tasks/slurm_logs.yml` | Collects Slurm controller, Slurm compute node, login node, and login-compiler node logs. | `slurm_controllers` (`stage: slurm_ctl`), `slurm_nodes` (`stage: slurm_node`), `login_nodes` (`stage: login_node`), `login_compiler_nodes` (`stage: login_compiler_node`) | Scheduler/cluster-service logs, job/runtime logs, OS/auth/system logs, bootstrap logs, DB/service logs (controller). | **Slurm Controller dirs:** `/var/log/slurm/`, `/var/log/rhsm/`, `/var/log/private/`, `/var/log/chrony/`, `/var/log/munge/`, `/var/log/insights-client/`, `/var/log/mariadb/` ; **Slurm Controller files:** `/var/log/cloud-init.log`, `/var/log/cloud-init-output.log`, `/var/log/ldms-cloudinit.log` ; **Slurm Node dirs:** `/var/log/slurm/` ; **Slurm Node files:** `/var/log/cloud-init.log`, `/var/log/cloud-init-output.log` ; **Login Node dirs:** `/var/log/slurm/`, `/var/log/chrony/`, `/var/log/private/` ; **Login Node files:** `/var/log/cloud-init.log`, `/var/log/cloud-init-output.log`, `/var/log/secure`, `/var/log/messages` ; **Login-Compiler Node dirs:** `/var/log/slurm/`, `/var/log/chrony/`, `/var/log/private/` ; **Login-Compiler Node files:** `/var/log/cloud-init.log`, `/var/log/cloud-init-output.log`, `/var/log/secure`, `/var/log/messages`. | Troubleshoots scheduler/queue behavior, auth (munge/secure), login path issues, node provisioning failures, time-sync/security anomalies, and Slurm control-plane service problems. |
| `omnia/log_collector/roles/log_collector/tasks/bundle.yml` | Builds final support bundle and metadata; computes SHA256; captures unreachable-node warnings; supports `curated_support` mode exclusions. | N/A | No new remote logs fetched; packages collected logs + `metadata.json` + checksum file. | Workspace root: `log_root` (default: `/opt/omnia/logs`), output bundle `omnia-logs-<identifier>-<timestamp>-IST.tar.gz`, checksum sidecar `.sha256`. **Curated support mode only:** Finds and removes files matching `exclude_patterns` from `vars/main.yml` recursively from `log_root` before bundling. Default patterns: `*.tmp`, `*.temp`, `*.bak`, `*.old`, `*stale*`, `*.gz`, `*.bz2`, `*.1-5`. Customize per environment. | Produces portable support artifact with provenance, integrity verification, and warning summary for triage/handover. |
| `omnia/log_collector/roles/log_collector/vars/main.yml` | Source-of-truth for all log collection paths and defaults. | N/A | Defines all collected directory/file targets by node type. | Variable lists for K8s, Slurm, login, login-compiler + workspace and PXE mapping path defaults. | Central place to tune scope of collection without changing task logic. |

| Output Location Pattern | What Gets Stored There |
|---|---|
| `{{ log_root }}/k8s/{{ inventory_hostname }}_{{ log_ts }}/` | All fetched K8s master/worker logs preserving source path structure. |
| `{{ log_root }}/slurm/{{ inventory_hostname }}_{{ log_ts }}/` | All fetched Slurm controller/node/login/login-compiler logs preserving source path structure. |
| `{{ log_root }}/metadata.json` | Run metadata (bundle name, timestamps, mode, warnings, host/user context). |
| `{{ log_root }}/omnia-logs-<identifier>-<YYYYMMDD-HHMMSS>-IST.tar.gz` | Final compressed bundle for sharing/support. |
| `{{ log_root }}/omnia-logs-...tar.gz.sha256` | SHA256 checksum for artifact integrity verification. |
